# UI_REVAMP_COMPONENT_SYSTEM

## Canonical Alignment
- Stage flow: `Job Creation -> Lot -> Images -> Final Pass -> Lab Testing -> Report -> Packing List`
- Roles: Production, R&D, Manager, Admin
- Core object: Lot
- Exclude Playground
- No KPI/dashboard clutter on operational screens
- Preserve traceability, permissions, validations, and auditability

## Pattern Library Summary
- Design direction: Zoho-inspired enterprise structure with light visual weight, strong hierarchy, and high scan speed.
- System goal: reusable Chakra-compatible patterns that can be applied across Job, Lot, Inspection, Lab Testing, Packet Management, Documents, and Exceptions.
- Layout principle: table-first lists, context-rich details, drawers for inline edits, and minimal modal usage.
- Visual principle: structured and business-like; avoid decorative dashboards and consumer-style components.

## Recommended Reusable Components and Layout Wrappers

### 1) App Shell
- `EnterpriseAppShell`
  - Left `SidebarNav`
  - Top `HeaderBar`
  - Main content frame with optional right context rail
- Use for all desktop role workspaces.

### 2) Sidebar
- `SidebarNav`
  - Workflow-first modules
  - Role-scoped visibility
  - Active state and compact collapsed mode
- Keep labels simple and business-facing.

### 3) Top Header
- `HeaderBar`
  - Global search trigger
  - Saved view selector
  - Role/company context
  - Session/user menu

### 4) Page Title + Breadcrumb
- `PageIdentityBar`
  - Title
  - Subtitle (optional)
  - Breadcrumb path
  - Key page status tag (optional)

### 5) Page Action Bar
- `PageActionBar`
  - Primary CTA (single dominant action)
  - Secondary actions
  - Bulk actions (when list supports multi-select)

### 6) Filter/Search Strip
- `FilterSearchStrip`
  - Saved view selector
  - Filter chips and quick filters
  - Contextual search input
  - Clear/apply controls

### 7) Enterprise Table Pattern
- `EnterpriseTable`
  - Sticky header support
  - Density modes (`compact`, `standard`)
  - Pinned columns (Job Number, Lot Number, Current Step)
  - Row click to detail
  - Row action menu for non-primary actions

### 8) Sticky Table Headers
- `StickyTableContainer`
  - Controlled vertical scroll area
  - Sticky `<Thead>` with elevated background and border

### 9) Detail Page with Tabs
- `DetailPageLayout`
  - Header section
  - `DetailTabs`
  - Main tab content
  - Optional right context rail

### 10) Right-Side Quick Edit Drawer
- `QuickEditDrawer`
  - Non-destructive inline editing
  - Preserve underlying page context
  - Submit/cancel and validation feedback

### 11) Compact Forms
- `CompactFormSection`
  - Required-first field order
  - Two-column desktop, one-column tablet/mobile
  - Inline validation and helper text only when action-relevant

### 12) Status Badges
- `StatusBadge`
  - Strict token mapping by state severity and actionability
  - No decorative badge variants

### 13) Timeline/History Block
- `HistoryTimeline`
  - Chronological audit/events feed
  - Actor, action, timestamp, optional note
  - Filter by event type and date range

### 14) Linked Records Panel
- `LinkedRecordsPanel`
  - Job/Lot/Sample/Packet/Dispatch/Documents cross-links
  - Always visible on relevant detail pages

### 15) Document Table
- `DocumentRegistryTable`
  - Required columns:
    - Doc Type
    - Job Number
    - Lot Number
    - Version
    - Status
    - Generated At
    - Generated By
  - Row-level actions:
    - `View PDF`
    - `Download Report PDF` or `Download Packing List PDF` (contextual)
    - `Share PDF`
    - `Print PDF`

### 16) Empty States
- `EnterpriseEmptyState`
  - Clear condition
  - Operational guidance
  - One primary recovery action

### 17) Exception Banners
- `ExceptionBanner`
  - Inline high-priority blocker or warning
  - Action text + resolve/escalate CTA
  - Dismiss only when non-blocking

## Where Each Pattern Should Be Used
- `EnterpriseAppShell`, `SidebarNav`, `HeaderBar`: every authenticated workspace route.
- `PageIdentityBar`, `PageActionBar`, `FilterSearchStrip`: all registry and queue pages.
- `EnterpriseTable`, `StickyTableContainer`: Job Registry, Lot Registry, Document Registry, Exception Queue, Packet list, trial tables.
- `DetailPageLayout`, `DetailTabs`: Job Detail, Lot Detail, Packet Detail, Exception Detail, Document Detail.
- `QuickEditDrawer`: row edit, metadata update, assignment change, minor record updates.
- `CompactFormSection`: Job Creation, Lot setup, Sample details, dispatch metadata forms.
- `StatusBadge`: stage status, readiness state, exception severity.
- `HistoryTimeline`: lot traceability and audit tabs.
- `LinkedRecordsPanel`: Lot Detail and Lot Traceability pages first; Job and Document detail second.
- `DocumentRegistryTable`: central Documents module and lot-level document tabs.
- `EnterpriseEmptyState`: all list and detail pages with no records.
- `ExceptionBanner`: blocker surfaces on task pages and exception details.

## Enterprise Interaction Rules
- Lists are table-first on desktop and tablet unless task-specific card mode is required.
- Row action edits open drawers by default.
- Modals reserved for destructive confirmation and full-screen PDF preview.
- Keep one primary CTA per page and per major task block.
- Preserve context: identifiers (Job Number, Lot Number, Current Step) stay visible at all times on detail pages.

## Anti-Patterns to Avoid
- KPI-heavy dashboards on operator execution pages.
- Decorative card grids that replace actionable tables.
- Mixed multi-task pages with competing CTAs.
- Hidden linkage between lot and downstream records.
- Technical/internal labels exposed to users.
- Modal-heavy editing flows that break context.
- Duplicate status wording across modules for the same concept.
- Unbounded form sections without required-first grouping.

## Chakra Compatibility and Maintainability Notes
- Build wrappers using Chakra primitives (`Box`, `Stack`, `Grid`, `Table`, `Drawer`, `Tabs`, `Badge`, `Alert`).
- Keep wrapper props narrow and composable.
- Centralize spacing, borders, font sizes, and status color tokens.
- Maintain consistent prop contracts for action bars, filters, and table row actions.

## Responsive Behavior Policy
- Desktop: full enterprise shell and table-first pages.
- Tablet: same IA, reduced density, split panes where useful.
- Mobile: task-completion surfaces only; condensed headers and primary action rails.

## Final Closure Notes
- `DetailTabsLayout` is the required tab shell for Inspection Job Detail, Packet Detail, and R&D Job Detail surfaces.
- Right rails on detail surfaces must use shared lineage/audit blocks:
  - `LinkedRecordsPanel`
  - `HistoryTimeline`
  - blocker/status panel where relevant
- Dispatch-readiness UX remains derived in this phase; persisted dispatch-document modeling is deferred to Phase-2 backlog.
